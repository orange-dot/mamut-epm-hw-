# Project Philosophy

## Internal EPM Framing

This philosophy now applies to `Mamut EPM` as a whole.

- `EPM1` carries it into the current playable software/runtime line
- `EPM2` carries it into the hardware program in this repo

The emphasis in this file is still most directly useful for `EPM2`, but the
stance is no longer hardware-only.

## Core Position

This is not a corporate hardware product exercise pretending to be the whole
story.

This project is about making a real instrument program for us. That means the
main question is not:

- what is the safest thing to ship?

The main question is:

- what kind of instrument is worth making at all?

## What We Are Optimizing For

We are optimizing for:

- a strong synth identity
- a sound that feels alive and specific
- a design language with character
- features that matter musically
- a machine we would actually want in the room

## What We Are Not Optimizing For First

We are not optimizing for:

- conservative scope just because it is safer
- corporate feature planning
- product management language
- lowest-risk architecture at all costs
- "good enough" if it means generic

## The Right Kind of Discipline

This does not mean chaos for the sake of chaos.

It means:

- bold choices
- strong taste
- intentional constraints
- building the weird thing on purpose
- keeping enough structure to finish a real synth

## Working Attitude

The desired mindset is:

- full hallucination energy
- zero interest in making another forgettable synth
- respect for engineering where it supports the instrument
- no worship of engineering purity when it weakens the instrument

## Practical Translation

In practice, this means:

- desktop format is still a smart move because it removes distraction
- polyphony is allowed even if it is harder
- unusual signal flow or modulation ideas are welcome
- the synth does not need to resemble a standard market segment
- technical decisions should serve the instrument's character

## Warning To Future Us

If the docs ever start sounding like a cautious internal planning deck, we have drifted away from the point of the project.
