# Product Direction

## Internal EPM Framing

`Mamut EPM` is now the umbrella program.

- `EPM1` = the current software line in `mamut-sint-sw`
- `EPM2` = the hardware line in `mamut-sint-hw`

This document defines product direction primarily for `EPM2`, while keeping the
shared `Mamut EPM` identity language aligned with `EPM1`.

## Working Goal

Define `EPM2` as a desktop analog polysynth that feels like a real instrument
with its own identity, not a generic market-friendly device.

The synth should prioritize:

- strong sonic identity
- fast performance workflow
- direct control surface design
- a point of view
- enough technical coherence to actually exist

## Current Direction

The current working concept is:

- desktop synth module
- no integrated keyboard in V1
- external MIDI keyboard as the primary controller
- analog signal path
- digital control layer

The immediate development setup assumes a `Kurzweil PC4` as the external keybed and MIDI controller.

The selected identity direction is now:

- `Experimental Performance Monster`

The selected tonal center is now:

- base: `Horizont + Pec`
- performance rupture layer: `Baklja`
- end-to-end macro axis: `Gravitacija`

## Why Desktop First

Desktop format is the fastest way to get to the interesting part.

It avoids the biggest non-sound distractions:

- sourcing and integrating a keybed
- key scanning hardware
- action feel validation
- larger enclosure and packaging complexity
- transport and mechanical durability issues

This allows the project to focus on:

- the voice architecture
- the sound
- panel design
- modulation workflow
- presets and MIDI behavior
- what makes this synth feel unlike the usual suspects

## Recommended V1 Shape

The current recommended V1 is:

- 6 voices
- 2 oscillators per voice
- sub oscillator
- one main filter per voice
- one VCA per voice
- envelope and modulation architecture controlled by the digital layer
- onboard presets
- MIDI DIN and USB-MIDI
- strong performance-layer macros
- scene morphing or equivalent large-gesture control
- deliberate voice individuality instead of perfect sterile sameness
- a sound architecture that can move from `Horizont` through `Pec` and into `Baklja`
- a global `Gravitacija` behavior that changes the organism of the sound, not just one block

Digital effects after the analog voice path are still a strong candidate for V1, not because they make the synth more market-complete, but because they may be part of the instrument's identity.

## Selected Direction: Experimental Performance Monster

This synth should behave like a performance machine, not a polite reference instrument.

The desired personality is:

- huge when opened up
- unstable in a useful way
- capable of elegant pads and brutal mutation
- immediate under the hands
- able to move far with one gesture

This direction implies a synth that rewards performance pressure:

- macros instead of only isolated parameters
- morphable states instead of static patches
- tension between precision and collapse
- controlled voice-to-voice variance

The goal is not "maximum weirdness". The goal is playable danger.

## Selected Tonal Center

The tonal center is no longer abstract.

It is now defined as:

- `Horizont` for scale, air, distance, and expensive width
- `Pec` for heat, density, low-mid authority, and pressure
- `Baklja` as the opened eruption in the instrument, accessed through performance controls rather than living permanently in the base tone

This matters because it defines the desired trajectory of performance:

- the synth should start from beauty and mass
- then gain stress and density
- then tear open into danger when pushed

That gives the instrument somewhere to go.

## Gravitacija As A System Behavior

`Gravitacija` is now the strongest candidate for the synth's end-to-end signature control.

It should not behave like a simple effect, spatial gimmick, or rotary simulation.

Instead, it should feel like the whole instrument is being loaded:

- more internal heat
- less headroom
- more saturation pressure
- more low-mid push
- more harmonic bending
- more psychological intensity

This is a major part of the synth's identity and may eventually shape both the analog output behavior and the performance macro system.

## Project Principles

### Instrument, not compliance exercise

Every major technical decision should be judged by whether it makes the synth more worth hearing and playing.

### Character first

The synth should not be reduced to a safe architecture because that is easier to explain. If a bolder choice gives the instrument a stronger voice, it deserves serious consideration.

### Direct panel over abstract flexibility

The synth should feel immediate. A smaller but well-chosen set of controls is better than a large menu tree or a feature list that slows down the player.

### Discipline in service of madness

We still need enough structure to finish something real. The goal is not random complexity. The goal is a focused machine with strange conviction.

## V1 Non-Goals

Do not treat the following as default V1 requirements just because they are conventional:

- integrated keyboard
- 8 or more voices
- multitimbral operation
- workstation sequencing
- maximal modulation matrix
- standard-market feature parity
- discrete-everything architecture just to prove purity

## Current Direction Questions

The high-level identity is chosen. What remains open is how this monster expresses itself.

Key follow-up questions:

- how dark versus bright should `Horizont + Pec` land in practice?
- how far should instability go before it stops being musical?
- how many large performance controls define the instrument?
- how much of `Gravitacija` lives in the analog layer versus the digital performance layer?
